🌍 Rozdział 1: Konfiguracja Serwera

W tym rozdziale dowiesz się, jak skonfigurować serwer Minecraft od podstaw, aby stworzyć stabilną i wydajną przestrzeń do gry online.

🛠️ 1.1: Wybór Typu Serwera

Przed rozpoczęciem konfiguracji musisz wybrać typ serwera: Vanilla (czysty Minecraft), Spigot (dla wsparcia wtyczek) lub Forge (dla obsługi modów). Każdy z nich oferuje różne funkcje. Wersja Vanilla jest najprostsza, podczas gdy Spigot i Forge wymagają dodatkowej konfiguracji, ale pozwalają na więcej możliwości dostosowania rozgrywki.

⚙️ 1.2: Pobieranie Plików Serwera

Przejdź na oficjalną stronę Minecraft i pobierz plik serwera. Wybierz najnowszą stabilną wersję lub wersję zgodną z Twoimi pluginami lub modami. Zapisz ją w folderze, który będzie służył jako główny katalog serwera.

💾 1.3: Uruchamianie Serwera Po Raz Pierwszy

Po pobraniu pliku serwera otwórz terminal lub wiersz poleceń, przejdź do folderu serwera i uruchom serwer za pomocą poniższego polecenia:

java -Xmx1024M -Xms1024M -jar minecraft_server.jar nogui

To polecenie inicjuje pliki konfiguracyjne oraz tworzy podstawowy świat serwera. Ustaw odpowiednie uprawnienia w systemie, aby serwer działał stabilnie.

🚀 1.4: Optymalizacja Ustawień JVM

Parametry JVM (Java Virtual Machine) mają znaczący wpływ na wydajność serwera Minecraft. Oto niektóre kluczowe ustawienia:

  • -Xmx – maksymalna ilość pamięci RAM dostępna dla serwera. Dostosuj ją do zasobów swojego serwera, np. -Xmx2048M dla 2 GB.
  • -Xms – minimalna ilość pamięci RAM. Zaleca się ustawienie jej na ten sam poziom co -Xmx dla stabilności.
  • -XX:+UseG1GC – zalecane ustawienie dla Minecrafta, gdyż algorytm G1GC lepiej radzi sobie z alokacją pamięci.

Przykładowe polecenie uruchamiające z optymalizacjami JVM wygląda następująco:

java -Xmx2048M -Xms2048M -XX:+UseG1GC -jar minecraft_server.jar nogui

📂 1.5: Tworzenie Skryptu Startowego

Aby uprościć uruchamianie serwera, możesz utworzyć skrypt startowy. Skrypt pozwala szybko uruchomić serwer i zastosować optymalne ustawienia bez ręcznego wpisywania polecenia za każdym razem.

Oto przykładowy skrypt dla systemu Windows:

@echo off java -Xmx2048M -Xms2048M -jar minecraft_server.jar nogui pause

A w systemach Linux lub macOS możesz utworzyć skrypt start.sh z poniższym kodem:

#!/bin/bash java -Xmx2048M -Xms2048M -jar minecraft_server.jar nogui

Pamiętaj, aby nadać uprawnienia do uruchomienia skryptu w systemie Linux/Mac za pomocą chmod +x start.sh.

🌐 1.6: Dostosowywanie Środowiska Uruchomieniowego

Jeśli planujesz uruchamiać serwer przez dłuższy czas lub udostępniać go większej liczbie graczy, warto skonfigurować środowisko uruchomieniowe. W szczególności:

  • Ustaw ograniczenia dostępu do plików serwera, aby tylko administratorzy mieli możliwość ich edycji.
  • Rozważ użycie narzędzi takich jak screen lub tmux na systemach Linux, aby zarządzać sesjami serwera w tle.
  • Ustaw regularne kopie zapasowe plików serwera, aby zapobiec utracie danych. Możesz skonfigurować automatyczny system backupu, np. przy użyciu cron na Linuxie.

📑 1.7: Konfiguracja Plików Pomocniczych

Serwer Minecraft generuje kilka plików konfiguracyjnych, które umożliwiają dostosowanie świata i zachowań serwera:

  • eula.txt – wymaga zaakceptowania warunków licencji EULA, zmieniając eula=false na eula=true.
  • banned-players.json – lista zbanowanych graczy, którą można edytować, aby ręcznie dodawać lub usuwać osoby.
  • whitelist.json – jeśli serwer działa w trybie whitelist, tylko gracze z tej listy będą mieli dostęp.

Zmieniając odpowiednie wartości w tych plikach, możesz dostosować serwer do własnych potrzeb i zapewnić większą kontrolę nad rozgrywką.
